# 📗 PM (Product Manager) 기획서 & 서비스 소개서: 기록하다 (Girokhada)

## 1. 제품 개요 (Product Overview)
**"부모님의 이야기를 AI가 기록합니다."**
**기록하다(Girokhada)**는 바쁜 자녀들을 대신해, 따뜻한 성품을 지닌 AI가 시니어(부모님, 조부모님)와 대화를 나누고 그분들의 생애를 인터뷰하여 가족만의 소중한 에세이, 자서전 형태의 기록물로 아카이빙(Archiving)해주는 AI 회고 서비스입니다.

### 🎯 제품 비전 (Vision)
- "기억이 사라지기 전에 가장 직관적인 방식으로 부모님의 삶을 기록한다."
- 디지털 소외 계층인 시니어들도 아무런 학습 없이 대화만으로 완성도 높은 기록을 남길 수 있게 돕는다.

### 👥 타겟 고객 (Target Audience)
1. **Primary User (발송자/관리자)**
   - 부모님의 지난 삶을 기록으로 남기고 싶고 소통하고 싶은 **3050세대 자녀**.
   - 부모님의 관심사에 맞는 주제/사진을 던져주고 답변을 받아보는 주체.
2. **End User (수신자/이용자)**
   - 스마트폰 조작이 서툴고 작은 글씨 타이핑이 불편한 **6080세대 시니어**.
   - 카카오톡 링크로 들어와 화면의 큰 버튼을 누르고 음성으로만 대답하는 주체.

## 2. 핵심 기능 정의 (Core Features)

### A. 시니어 맞춤형 인터뷰 세션 (End User UX)
철저히 디지털 소외 계층(시니어)의 눈높이에 맞춘 'Zero-Learning' 인터페이스입니다.
1. **One-Click 링크 진입 (No-Login)**
   - 앱 설치나 회원가입 시나리오 일절 배제. 자녀가 카카오톡으로 보낸 고유 세션 링크를 클릭하면 즉시 인터뷰가 시작됩니다.
   - 대상자 개인의 이름(ex. "김순자님, 안녕하세요")이 호명되며 따뜻하게 맞이합니다.
2. **Voice-First 인터페이스 (초대형 마이크 버튼)**
   - 텍스트 입력창은 최소화하고 화면 중앙에 **거대한 마이크(🎙️) 버튼**을 전면 배치하여 직관적인 조작을 유도합니다.
   - **STT(Speech-to-Text) 자동화**: 마이크를 끄면 사용자의 음성을 즉시 텍스트로 변환하고 시스템에 전송합니다.
3. **'손주' 페르소나의 유기적 꼬리 질문 (LLM Prompting)**
   - 기계적인 문답이 아닌, **사용자 답변의 구체적 디테일을 캐치하여 되묻는** 높은 수준의 공감 능력을 시뮬레이션합니다. (ex. "첫 월급으로 내복 사드렸어" ➡️ "아, 부모님 내복이요! 그때 부모님 표정이 어떠셨어요?")
   - 적절한 길이 고정: 피로도를 막기 위해 딱 3~4회의 핑퐁 턴만 진행하여 한 편의 짧은 회고록 단위로 마무리됩니다.
4. **다양한 형태의 자극 (Memory Stimulator)**
   - 단순 질문뿐만 아니라, **옛날 사진(추억) 기반 질문**이나 **빈칸 채우기** 모드를 지원하여 과거의 기억을 자연스럽게 꺼낼 수 있도록 유도합니다.

### B. 관리자 대시보드 (Admin / Primary User 용)
자녀가 부모님의 인터뷰를 관리하고 결과물을 확인하는 뒷단 시스템입니다.
1. **맞춤형 인터뷰 링크 발급기**
   - 대상자 이름과 질문 방식을 선택하고 저장하면 고유한 ID가 담긴 링크가 1초 만에 발급됩니다.
   - **사진 파일 업로드 기능**: 부모님의 옛날 사진, 가족 여행 사진 등을 직접 업로드(Supabase Storage)하여 질문의 배경으로 띄울 수 있습니다.
2. **대화 기록 아카이브 및 원본 보기**
   - 부모님이 인터뷰를 통해 남기신 모든 발화(텍스트/음성 변환 텍스트)를 시간순으로 실시간 확인 가능합니다.
   - 사진 기반 질문의 경우 썸네일에 마우스를 올리면 원본 사진(Lightbox)을 크게 띄워 볼 수 있습니다.
   - 긴 대화라도 글자가 잘리지 않고 편하게 읽을 수 있는 카드형 뷰어가 제공됩니다.
3. **안전한 데이터 관리 (Soft Delete)**
   - 실수로 질문/답변 카드를 삭제하더라도 치명적인 데이터 유실을 막기 위해 1차적으로 '소프트 삭제(비활성화)' 처리되어, 서비스 차원에서 원본 복구가 가능하도록 DB 안정성을 고려했습니다.

## 3. 기술 스택 (Tech Stack) 및 구조
- **Frontend**: Next.js (App Router), React, Tailwind CSS
- **Database & Storage (BaaS)**: Supabase (PostgreSQL, Storage, RLS)
- **AI Integration**: OpenAI (GPT-4o 계열 활용 텍스트 생성, Whisper 활용 STT, TTS)
- **배포 및 인프라 환경**: Vercel 연동 고려 (서버리스 구조)

## 4. 향후 로드맵 (Next Steps for PM)
- **Phase 4**: 모인 3~4개의 짧은 회고록(Session)을 하나로 묶어 LLM이 매끄러운 한 편의 '수필 형식 자서전'으로 요약·가공해주는 리포트 생성 기능 도입.
- **Phase 5**: 생성된 자서전을 자녀가 PDF나 웹 매거진 형태로 쉽게 다운로드하고 가족 친지들과 카카오톡으로 공유할 수 있는 Share 기능 제공.
- **Phase 6**: 어르신이 텍스트를 읽는 대신 직접 귀로 들을 수 있는 수준 높은 TTS(음성 합성 모델) 자동 재생 완전성 확보.
